# 前借り君（利息のみ支払い・元本据え置き型）整理メモ

目的: 経営陣向けに、RBF（Revenue Based Financing）と比較しつつ「利息だけを払い続ける」「元本は据え置きで借り続ける」モデルの可否と留意点を体系的に説明するための素案。

前提: 対象は法人と個人事業主（事業者扱い）。完全な個人は対象外。

※ 本メモは一般的な整理であり、最終判断は弁護士・貸金業登録の専門家に確認が必要。

---

## 1. 用語の整理
**前借り君のアイデア**  
- 元本は返済せず据え置き（長期・継続）  
- 利息（または利息相当額）だけを継続的に支払う  
- 実務的には「利息のみローン」「ロールオーバー型」に近い構造

**RBFとの対比**  
- RBF: 売上の一定割合で元本とリターンを回収し、一定回収で契約終了  
- 前借り君: 元本が減らないため、支払いは一定だが終了条件が曖昧

---

## 2. キャッシュフロー図（スライド用・視覚注記つき）

### 2-1. 通常のローン
```
時間 → →
借入:   ● +100
返済:   ■ -X  ■ -X  ■ -X  ■ -X  ■ -X
残高:   100 → 80 → 60 → 40 → 20 → 0
```

### 2-2. RBF（売上連動・回収上限あり）
```
時間 → →
借入:   ● +100
返済:   ▲ -売上%  ▲ -売上%  ▲ -売上%  ▲ -売上%
残高:   100 → 70 → 50 → 25 → 0（回収上限で終了）
```

### 2-3. 前借り君（利息のみ・元本据え置き）
```
時間 → →
借入:   ● +100
返済:   ◆ -利息  ◆ -利息  ◆ -利息  ◆ -利息
残高:   100 → 100 → 100 → 100 → 100
```

スライド注記案:
- ●=初回借入、■=定額返済、▲=売上連動返済、◆=利息のみ  
- 残高ラインが「減らない」ことが最も直感的な違い  
- 収益（利息）は安定だが、元本回収は別途設計が必要
- 色分けルール案: ●=濃紺、■=グレー、▲=オレンジ、◆=緑、残高ライン=青  
- 凡例案: 「支払いの性質」を色で区別（固定/連動/利息のみ）

**簡略図パターン（スライド差分用）**
```
パターンA（線のみ）
通常:   借入┈返済→残高0
RBF:    借入┈売上連動→上限で終了
前借り: 借入┈利息のみ→残高据置

パターンB（記号のみ）
通常:   ● ■ ■ ■ ■
RBF:    ● ▲ ▲ ▲
前借り: ● ◆ ◆ ◆
```

## 3. 法規制・制度面の論点（共通）
**論点1: 「貸付」に該当するか**  
- 元本を渡し、利息相当の対価を受け取るなら「金銭消費貸借」に該当する可能性が高い  
- 貸金業法の登録要否、利息制限法・出資法の上限金利が論点

**論点2: 「利息のみ支払い」の契約設計**  
- 返済期限をどう設定するか（期限の定めがない場合の法的扱い）  
- 更新条項、期限の利益喪失条項、満期時の扱い  
- 永続性を謳うと規制・消費者保護のリスクが高まる

**論点3: 実質年率と上限金利**  
- 形式上「手数料」でも実質利息に該当しうる  
- 実質年率の計算と上限金利の遵守が最重要

---

## 4. ターゲット別の論点分岐（法人 / 個人事業主）

### 4-1. 法人向け
- 事業者向けでも貸金業法の対象になり得る  
- 取引慣行として「更新・借換え」をどう運用するかが焦点  
- 返済能力評価・担保/保証の設計が実務上の肝
- 登録/許認可: 貸金業登録が必要になる可能性が高く、スキーム次第で例外の有無を確認

### 4-2. 個人事業主向け（事業者扱い）
- 事業者として扱う前提でも規制は残る  
- 返済期限・説明義務・適合性の観点は重要  
- 広告表現（「元本不要」「永久」など）のリスクが高い
- 登録/許認可: 貸金業登録のほか、表示・説明・勧誘規制の影響が大きい

**法人/個人事業主の論点まとめ（スライド向け表）**
```
観点            | 法人向け                         | 個人事業主向け
----------------|----------------------------------|----------------------------------------
規制の強さ       | 相対的に低めだが適用あり         | 相対的に高め（説明・適合性が重要）
登録/許認可      | 貸金業登録が必要になる可能性      | 貸金業登録＋表示/勧誘/説明規制が強い
期限/更新設計    | 更新・借換え運用が焦点           | 期限・説明義務・適合性が焦点
広告表現リスク   | 誤認・優良誤認に注意             | 「元本不要」「永久」表現は高リスク
審査/担保        | 返済能力評価・担保/保証の設計     | 返済能力評価・生活影響の確認が重い
```

---

## 5. 事業面のメリット/デメリット（RBF比較）

### 借り手側のメリット
- 元本返済が不要なため資金繰りが安定  
- 返済負担の予見性が高い（利息固定の場合）

### 借り手側のデメリット
- 総コストが長期化すると高くなる  
- 終了条件が曖昧になりがち（出口設計が重要）

### 貸し手側のメリット
- 収益が長期化し、安定する可能性  
- 元本回収のタイミングを設計できる

### 貸し手側のデメリット
- 元本回収リスク（借り手破綻時の回収困難）  
- 長期契約に伴う規制コスト・信用リスク

---

## 6. 「可能ならどうやるか」設計パターン案
※ いずれも規制適合の精査が必須

### 案A: 利息のみ返済＋満期で元本一括（バルーン）
- 表向きは「期限あり」  
- 満期ごとに更新・借換えを行う設計  
- 規制的には通常の貸付と同様

### 案B: 利息のみ返済＋元本償還権（任意償還）
- 借り手が任意に元本返済できる条項  
- 事実上の据え置きだが、出口が用意される

### 案C: 売上連動の「利息相当」支払い（RBF寄り）
- 利息と売上連動を組み合わせる  
- 実質金利の整理が難しいため慎重設計

---

## 7. 既存事例の調査項目（調べるべき観点）
- 国内: 銀行・信金・ノンバンクでの「利息のみ期間」「据置期間」商品  
- 海外: Interest-only loan / Balloon loan / Revolving credit の規制枠組み  
- 契約条件: 期限、更新条件、利率見直し、担保/保証  
- 規制: 金利上限、説明義務、適合性、広告規制  
- 失敗/炎上事例: 取引継続リスク、消費者トラブル

---

## 8. 経営陣向けに強調したいポイント
- 「利息だけ払い続ける」設計は基本的に貸付規制の枠内  
- 永続性を謳うと規制リスク・消費者保護リスクが高まる  
- 実質金利の計算と上限金利の遵守が最大のハードル  
- 事業者向けでも貸金業登録や利息制限が問題になりうる

## 11. 規制論点スライド（結論先出し）
**結論**: 「利息のみ支払い」型でも原則は貸付に該当し、貸金業法・利息制限の枠内で設計が必要。

**理由（要点のみ）**  
- 元本提供＋対価受領は金銭消費貸借に該当しやすい  
- 実質年率の上限遵守が不可避  
- 永続性・誤認を招く表現は規制リスクが高い

**設計の肝**  
- 期限・更新・出口（任意償還/満期）を明確化  
- 広告・説明を「誤認なし」に設計  
- 法人/個人で規制強度が大きく異なる

## 12. スライド原稿（高校生語彙・3分想定）

### 1枚目: 今日のテーマ
**タイトル**: 前借り君（利息だけ払い続ける仕組み）はできるか  
**一言**: RBFと違って「元本が減らない」モデルを検討する

### 2枚目: まずは図で理解
**ポイント**:  
- 通常ローン = 元本が減る  
- RBF = 売上に合わせて減る  
- 前借り君 = 元本が減らない

### 3枚目: いちばん大事なルール
**一言**: これは「貸付」になりやすい  
**意味**: 貸金業法や利息制限のルールの中でやる必要がある

### 4枚目: 許認可があるとできること
**結論**: 貸金業登録がある前提で設計できる範囲が広がる

**簡単整理（貸し手側）**
```
状態                         | できること（目安）
----------------------------|------------------------------
登録なし                     | 原則として貸付スキームは難しい
登録あり                     | 貸付として設計可能（上限金利遵守が必須）
登録あり＋厳格な説明/広告運用 | 事業者向けで運用可能性が高まる
```
※ 例外の有無は個別確認が必要

### 5枚目: 事業者向けでも注意点
**ポイント**:  
- 事業者向けでも規制はある  
- 「永久」「元本不要」などの表現は危険  
- 期限と出口は必ず作る

### 6枚目: 進め方（現実的な道）
**おすすめ**: B寄り（登録前提で設計を詰める）  
**ただし**: 登録が取れない場合はA（事実上むずかしい）になる

### 7枚目: 次のアクション
- 貸金業登録の取得可否を最優先で確認  
- 期限・更新・出口の設計を先に決める  
- 実名事例を調査して比較スライドに埋める
